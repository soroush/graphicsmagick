ImageMagick 5.1.0, January 1, 2000

  • ImageMagick 5.0 is a major revision. A number of changes were made to the structures and API. You can expect no further, or at most very minor, changes to the API in the future. PerlMagick and Magick++ were updated to reflect the changed API

  • Added a memory and disk cache. By default, any image or set of images that consume more than 80mb of memory are cached to disk to conserve memory.

  • Changed the parameters to the CreateImage and GetPixel methods to make the routines more robust. These methods are convenience routines to eaily transfer pixel data into and out of ImageMagick from a C API.
