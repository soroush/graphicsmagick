ImageMagick now depends on the external files, delegates.mgk and magic.mgk.
If ImageMagick commands fail, verify that these files are installed
and valid for your environment. For details beyond this document, see
README.txt and ImageMagick.html.

Unix:

  Download

    ftp://www.wizards.dupont.com/pub/ImageMagick/ImageMagick-5.2.0.tar.gz

  and type

    gunzip -c ImageMagick-5.2.0.tar.gz | tar xvf -
    cd ImageMagick-5.2.0
    ./configure
    make
    make install
    identify -verbose logo:

  Alternatively, download the appropriate binary from
  ftp://www.wizards.dupont.com/pub/ImageMagick/binaries.

  If you download the binary distribution, you must move
  the file share/ImageMagick/delegates.mgk to the directory
  /usr/local/share/ImageMagick/ or to the directory $HOME/.magick
  and move the file share/ImageMagick/magic.mgk to directory
  /usr/local/share/ImageMagick/, or to the directory $HOME/.magick.
  If the binary distribution is anywhere but /usr/local you will
  probably need to set the LD_LIBRARY_PATH, MAGICK_DELEGATE_PATH, and
  MAGICK_MODULE_PATH environment variables.  For example,

    setenv LD_LIBRARY_PATH ~/lib
    setenv MAGICK_DELEGATE_PATH ~/share/ImageMagick
    setenv MAGICK_DELEGATE_PATH ~/ImageMagick/modules/coders


  In some cases, ImageMagick needs to write temporary files.  If your
  default location is too small, use the TMPDIR environment variable to
  specify a path. For example:

    export TMPDIR=/home/tmp


Win2K, NT, Win95, Win98:

  Download

    ftp://www.wizards.dupont.com/pub/ImageMagick/win2k/ImageMagick-win2k.zip

  and double click on it to extract the distribution.  Next, select
  File->New->Folder to create c:\ImageMagick.

    ImageMagick-5.2.0/delegates/win2k.mgk

  to

    c:\ImageMagick\delegates.mgk

  Use Copy/Paste/Rename to copy and rename win2k.mgk to delegates.mgk.  Now
  copy

    ImageMagick-5.2.0\delegates\magic.mgk

  to

    c:\ImageMagick\magic.mgk

  If you have an X server, change to directory
  ImageMagick-5.2.0/ImageMagick-5.2.0/bin and double-click on
  display.exe.  Otherwise, select Start->Programs->Command Prompt.  In
  the Command Prompt window type

    cd ImageMagick-5.2.0
    identify -verbose logo:

  Alternatively, download the ImageMagick NT source

    ftp://www.wizards.dupont.com/pub/ImageMagick/win2k/ImageMagick-5.2.0.zip

  and double click on the ImageMagick Visual C++ project to build from
  the source.

  In some cases, ImageMagick needs to write temporary files.  If your
  default location is too small, use the TMPDIR environment variable to
  specify a path. For example:

    set TMPDIR=d:\temp


MAC:

  Download

    ftp://www.wizards.dupont.com/pub/ImageMagick/mac/ImageMagick-mac.sit.hqx

  and double click on it to extract the distribution.   Move to the
  ImageMagick-5.2.0 directory and double click on the identify program
  icon.  Next, type

    -verbose logo:

  in the command window and choose File->Quit to exit.

  Alternatively, download the ImageMagick MAC source

    ftp://www.wizards.dupont.com/pub/ImageMagick/mac/ImageMagick-5.2.0.sit.hqx

  and double click on the Make AppleScript script to build from the source.


VMS:

  Type

    unzip ImageMagick-5.2.0.zip
    set default [.imagemagick]
    @make
    identify -verbose logo:
